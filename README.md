# 萬象命書 FateVerse

> 一次看懂東西方命理

萬象命書是一個完全以前端執行為主的文化探索、娛樂與自我反思工具。它把八字、紫微斗數、西洋星盤、生肖、生命靈數、姓名基礎資料與籤詩辨識放在同一個介面中，讓使用者觀察不同文化模型如何描述同一個人。

生日、姓名與圖片都在瀏覽器中處理。網站不需要登入、後端或 API Key；本地 AI 是使用者主動選擇的增強功能，沒有 WebGPU 時仍可產生完整規則式報告。

## 功能

- 探索命盤：公曆／農曆、四柱、藏干與參考比例、月令旺相休囚死、十神、十二長生、旬空、胎元／胎息／命宮／身宮、八組大運，以及主干支的五合、六合、六沖、六害、六破、刑、三合與三會偵測。
- 五行分析：八個天干地支主五行的數量、比例與相對強弱。
- 日主強弱與喜用神：以公開計分規則（月令加成、天干計分、藏干比例、月支加權）判定強／偏強／中和／偏弱／弱，附完整計分明細；依扶抑法加基本調候給出喜用五行與白話生活對應（顏色、方位、習慣），以及近三年流年干支的十神類別速覽。
- 紫微斗數：十二宮、命身主、五行局、主輔星、亮度、四化、可選日期的大限／流年／流月／流日與命宮三方四正；可切換通行／中州安星、年界與晚子時規則。
- 西洋星盤：十個星體的地心黃道位置、太陽／月亮星座、逆行與主要相位；提供十星元素／模式分布、相位緊密度，並在有經緯度時加入上升、等宮制／整宮制十二宮、角續果宮與人生領域比較。SVG 輪盤採傳統排列（有上升時 ASC 在左），含星座符號分色、度數刻度、宮首線、行星符號防重疊與相位分色。
- 生命靈數：保留 11、22、33 大師數，顯示完整加總過程。
- 姓名分析：750+ 個精選姓名用字的字義與簡化五行對照，加上 Unicode Unihan 資料庫的全漢字覆蓋（U+4E00–U+9FFF 共 20,992 字的總筆畫與部首五行對照）——任何姓名都能算出五格剖象（天／人／地／外／總格）、81 數理吉凶（常見通行版本）與三才五行關係；筆畫優先採精選字庫計法、Unihan 補齊，均可手動修正。
- 塔羅牌：22 張大阿爾克那三牌陣（過去／現在／未來，含正逆位），以及依生日計算的人格牌與靈魂牌；生日塔羅同時併入報告的系統結論。
- 全面整合剖面：把八字、西洋星盤、生肖、生命靈數、生日塔羅、紫微五行局、姓名用字與手相手型全部換算到同一套五行座標，做「權重化」加權平均（八字權重最高、其餘依資訊量遞減），輸出一張最完整的整體五行剖面，並列出每套系統各自貢獻的主向與依據、整合完成度，以及尚未接上的系統與前往連結。
- 融合解讀：在整合剖面之上，把八字、五行、生肖、西洋星座、生日塔羅、紫微五行局、姓名用字與手相手型（有做時）做元素共識投票、四條跨系統性格光譜（生日塔羅與手相亦納入）、系統 × 面向雷達總覽（每套系統在四個性格面向上的立場一次看，可單選比較）、個性／工作／感情／身心四個領域的白話解讀、系統間共識與矛盾亮點，以及八字大運與紫微大限的時運交叉比對；全程口語化說明，並清楚標示跨文化對照僅為近似。
- 綜合報告：規則式 fallback 永遠可用；可選擇由瀏覽器本地 WebLLM 重新整理。
- 拍籤解籤：圖片縮放、Canvas 預處理、繁體中文 OCR、Fuse.js 模糊比對及主題解讀。
- 決策儀式「擲」：卡在兩難時的自我覺察工具，不預測未來。以命盤特徵為骰種擲一次隨機的「動／靜」命運骰，先問你「希望是哪一面」，揭曉後問你的第一情緒反應（鬆一口氣／失望／沒感覺）——用投射原理把你早已擁有的答案照出來，並給一張打破僵局的行動卡與可下載的 Canvas 分享圖；歷史只存本機（IndexedDB），可隨時清除。策略依據見 `docs/strategy-research.md`。
- 巴納姆效應鏡子：反過來教你拆解算命話術的單元。把使用者的真實命盤洞察與隨機挑出的通用「巴納姆句」混合成兩組盲測，猜猜哪組才是真的；揭曉後標示每句通用句用了哪種冷讀術手法（彩虹式話術、精準奉承、巴納姆句、模糊泛用、安全提問、高機率猜測、次分類話術）並附技巧圖鑑。沒有命盤時可用固定示範資料體驗，全程本地計算。
- 拍手相：上傳或拍攝手掌照片後，於瀏覽器端自動影像分析（YCbCr 膚色偵測、手型與手指比例量測、掌心 Sobel 邊緣偵測分區統計），自動判讀手型（土風水火）與生命線、智慧線、感情線、事業線特徵並標示信心度；判讀結果可逐項手動修正，照片不上傳、分析失敗時退回手動指認。
- 今日指引：66 張本地靜態卡；日期固定抽卡，也可重新抽取。
- 本地資料管理：預設不永久保存出生資料，可選擇保存最近一次分析並隨時清除。
- PWA：應用程式 shell、基礎命理資料、示範籤詩與今日指引可離線使用。
- 品牌化 Open Graph／X 社群分享預覽圖。
- 輕量動態介面：頁面與命盤進場、今日卡重抽、資料條動畫、手機選單與桌面卡片回饋；自動遵守 `prefers-reduced-motion`。

## 技術架構

- React 19、TypeScript strict mode、Vite
- Tailwind CSS、React Router（HashRouter）、Zustand
- `lunar-javascript`：農曆與四柱計算
- `iztro`：紫微斗數通行排法盤面
- `astronomy-engine`：行星、月亮、黃道位置與恆星時計算
- `tesseract.js`：使用者主動啟動的繁體中文 OCR
- `fuse.js`：籤詩模糊比對
- `idb-keyval`：使用者同意後的裝置端資料保存
- `@mlc-ai/web-llm`：使用者主動啟用的瀏覽器本地模型
- Zod：WebLLM JSON 輸出驗證
- Vitest、React Testing Library、ESLint、Prettier

命理計算與文字解讀嚴格分離：

```text
使用者輸入 → JavaScript／命理套件計算 → FateReportInput JSON
           → 規則式模板（預設）或 WebLLM（可選）→ 報告
```

AI 不重新計算四柱、生肖、星座、生命靈數，也不能補猜未提供資料。

## 本地開發

需求：Node.js 22（GitHub Actions 也使用 Node.js 22）。

```bash
npm install
npm run dev
```

品質與建置指令：

```bash
npm run typecheck
npm run lint
npm run test
npm run build
npm run preview
```

Vite 開發環境使用 `/`；正式建置的 base path 是 `/FateVerse/`，對應 `donald5043/FateVerse`。

## GitHub Pages 部署

`.github/workflows/deploy.yml` 會在 `main` 更新時依序執行：

1. checkout 與 Node.js 22 設定。
2. `npm ci`。
3. `npm run lint`、`npm run test`、`npm run build`。
4. 上傳 `dist` Pages artifact。
5. 透過 GitHub Pages 官方 Action 部署。

在 repository 的 **Settings → Pages → Build and deployment → Source** 選擇 **GitHub Actions**。合併至 `main` 後，正式網址預期為：

```text
https://donald5043.github.io/FateVerse/
```

網站採 HashRouter，所以功能頁網址為 `https://donald5043.github.io/FateVerse/#/profile`，重新整理不會 404。

## WebLLM 注意事項

- 網站初始化不會下載模型；只有設定頁按下「同意並啟用本地 AI」才開始。
- 預設模型是 WebLLM 預建清單中的 `Qwen2.5-0.5B-Instruct-q4f16_1-MLC`；原 Qwen3 0.6B 因 iOS 記憶體與隱藏思考成本不再作為預設。
- 模型 repository 檔案合計約 276 MB；WebLLM 標示約需 945 MB GPU 記憶體，實際下載與快取仍依瀏覽器版本不同。
- 建議至少 4 GB 系統記憶體與約 1.5 GB 可用 GPU 記憶體。
- 模型不放在 Git repository；由 WebLLM 指定來源下載至瀏覽器快取。
- 支援進度、取消與清除模型快取。部分瀏覽器可能要等下一個載入回呼才完成取消。
- 模型回傳必須通過 Zod JSON schema；解析、載入或生成失敗時會自動回到規則模板。
- 模型下載完成後會先執行 8 token 非 JSON 健康檢查；未真正回傳文字就不會顯示「已就緒」。
- iPhone／iPad 使用 140 token 短篇增強，再與完整規則報告合併；60 秒沒有完成會中止並重設 Worker。其他裝置上限 180 token／90 秒。
- 首 token、串流停滯與總時間各有 watchdog；錯誤會顯示 `AI-Lxx`／`AI-Gxx` 診斷碼，不再靜默維持舊報告。
- AI 回傳的摘要與兩項建議顯示在獨立的「本地 AI 生成內容」卡；規則摘要、命盤數值與 AI 原文明確分開，不再互相覆寫。
- 手機 WebGPU、記憶體與散熱差異很大。輕量模式包含全部精確排盤與文化內容，不是錯誤狀態。

## OCR 注意事項

- 支援 JPG、PNG、WebP，單檔上限 15 MB；最長邊先縮至 1800 px。
- Canvas 模式包含原圖、灰階、高對比、黑白二值化、旋轉 90 度與中央裁切。
- `tesseract.js` 只在按下 OCR 後延遲載入，預設語言為 `chi_tra`。
- 首次 OCR 通常需要網路下載語言資料。PWA 不預先快取 OCR 語言模型。
- OCR 結果一定要由使用者校對，不會直接當成正確籤文。
- 第一版主要支援繁體印刷體；手寫、反光、傾斜或複雜背景會降低辨識率。

## 籤詩資料格式

資料位於 `public/data/fortune-sticks/`，每筆遵循 `src/types/fate.ts` 的 `FortuneStick`：

```json
{
  "id": "unique-id",
  "system": "sixty-jiazi",
  "sourceName": "資料集名稱",
  "number": 1,
  "title": "籤名",
  "level": "中吉",
  "poem": ["第一句", "第二句"],
  "story": "經查證的典故或明確的無典故說明",
  "summary": "自行撰寫的現代摘要",
  "interpretations": { "overall": "整體解讀", "career": "工作解讀" },
  "actions": ["可選擇的行動"],
  "risks": ["需要留意的風險"],
  "keywords": ["比對關鍵字"],
  "dataSource": { "sourceName": "來源", "license": "授權", "notes": "備註" }
}
```

### 新增籤詩資料

1. 確認原文為公版或已取得明確授權；不要複製授權不明網站的現代解釋。
2. 加入穩定且唯一的 `id`，並正確標示 `system`、來源、授權與備註。
3. 原文、典故與現代摘要分開；無法驗證典故時留空或明確標示未知。
4. 把資料加入對應 JSON；`vite-plugin-pwa` 會在 production build 自動把符合規則的 JSON 納入 precache，不需手動維護 service worker 清單。
5. 為完整文、OCR 錯字、籤號與查無結果加入 matcher 測試。

目前兩份籤詩 JSON 各收錄 12／10 筆 FateVerse 自編七言四句籤（含九面向解讀、行動與風險），另有使用者照片逐字轉錄的可追溯樣本；所有自編籤以 CC BY 4.0 授權，仍不是完整的廟宇正式籤本。

## 擴充基礎資料

### 生命靈數

計算規則與內容位於 `src/engines/numerology-engine.ts`；新增內容時保留 `public/data/numerology.json` 的支援數字契約同步，並加入一般數字、大師數與計算過程測試。

### 今日指引卡

在 `public/data/daily-guidance.json` 加入符合 `DailyGuidanceCard` 的項目：`id`、`title`、`keyword`、`message`、`reflectionQuestion`、`suggestedAction`。維持至少 30 張並避免預言、醫療或投資斷言。

### 姓名字典

精選字庫內建 750+ 個常用姓名用字（`src/data/name-dictionary.ts`），全漢字筆畫與部首由 `src/data/unihan-char-data.ts`（Unihan 自動產生，見 `scripts/generate-unihan-data.py`）補齊，含字義、簡化五行對照與現代標準筆畫；筆畫計法有流派差異的字（如艸部）刻意不提供筆畫。新增筆畫資料時必須標示來源類型；沒有完整康熙字典資料時不可標成正式康熙筆畫。

## 隱私

- 沒有後端、帳號、分析資料庫或付費 API。
- 出生資料預設只在分頁記憶體中，重新整理後消失。
- 只有使用者在設定頁開啟「分析完成後保留在此裝置」，最近一次結構化結果才寫入 IndexedDB。
- OCR 圖片不寫入 IndexedDB；WebLLM prompt 只交給裝置端模型。
- OCR 語言資料和 WebLLM 模型的下載會連線到套件／模型指定的公開內容來源，但不包含姓名、生日、圖片或報告資料。
- 決策儀式的歷史紀錄只存在本機 IndexedDB（僅含你輸入的問題與選擇，不含出生資料），可在儀式頁一鍵清除，設定頁「清除全部資料」也會一併移除。
- 設定頁可清除偏好、最近分析、網站快取與模型快取。

## 免責聲明

萬象命書 FateVerse 提供的內容，僅供文化探索、娛樂與自我反思。所有命理、籤詩、星座、姓名與數字分析，均不代表確定的未來，也不應作為醫療、法律、投資、財務或重大人生決策的唯一依據。重要決定請依照實際情況，並諮詢具專業資格的人員。

本網站不提供醫療診斷或治療建議。如有健康疑慮，請尋求合格醫療專業人員協助。

## 已知限制

- 八字沒有套用真太陽時；大運採 lunar-javascript sect 2，流派差異需另行比較。
- 五行主圖只統計四柱天干與地支主五行；藏干比例與旺相休囚死另列為固定參考表，尚未完成日主強弱、格局、調候、喜用神與化氣條件。
- 西洋星盤已提供等宮制與整宮制的落宮及宮位主題比較，尚未加入 Placidus 等不等宮制、宮主星與飛星；未提供經緯度時不計算上升與宮位。
- 紫微斗數可比較通行／中州安星及部分分界設定；不同流派的四化表、星曜亮度、閏月規則仍可能不同。
- 姓名字典與籤詩皆為少量示範資料；姓名五格不計算，只保留 Beta 介面。
- OCR 重新裁切第一版採每次裁去四周 5%，尚無拖曳式裁切框。
- WebLLM 無法在沒有 WebGPU 或記憶體不足的裝置使用，且小模型文字品質可能波動。
- PWA 不預先快取大型模型或 OCR 語言資料。

## Roadmap

長期產品定位與跨領域策略研究見 [docs/strategy-research.md](docs/strategy-research.md)（決策儀式、自我敘事引擎、投射式鏡子等重新定位方向，以及零後端分享、生成藝術命盤指紋等構想）。以下為近期工程項目：

- 引入經授權、可追溯版本的完整籤詩資料集。
- 拖曳式圖片裁切、透視校正與多語 OCR。
- 正式康熙字典資料與可驗證的筆畫來源。
- 加入西洋星盤 Placidus 等不等宮制、相位權重與可切換的熱帶／恆星黃道研究。
- 擴充紫微流派設定為不同四化表、亮度表與閏月規則的可比較模式。
- 可選的真太陽時流程，清楚顯示地理、時區與校正差異。
- Web Worker 化更重的影像處理與本地模型推論介面。
- 無障礙與多裝置實機測試矩陣。
